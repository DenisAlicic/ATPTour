  \href{https://github.com/webpack/webpack}{\texttt{  }} \doxysection*{Sass Loader}

 

Loads a Sass/\+SCSS file and compiles it to CSS.

 

Use the \href{https://github.com/webpack-contrib/css-loader}{\texttt{ css-\/loader}} or the \href{https://github.com/webpack-contrib/raw-loader}{\texttt{ raw-\/loader}} to turn it into a JS module and the \href{https://github.com/webpack-contrib/mini-css-extract-plugin}{\texttt{ mini-\/css-\/extract-\/plugin}} to extract it into a separate file. Looking for the webpack 1 loader? Check out the \href{https://github.com/webpack-contrib/sass-loader/tree/archive/webpack-1}{\texttt{ archive/webpack-\/1 branch}}.

\doxysubsection*{Install}


\begin{DoxyCode}{0}
\DoxyCodeLine{npm install sass-\/loader node-\/sass webpack -\/-\/save-\/dev}

\end{DoxyCode}


The sass-\/loader requires \href{https://github.com/webpack}{\texttt{ webpack}} as a \href{https://docs.npmjs.com/files/package.json\#peerdependencies}{\texttt{ {\ttfamily peer\+Dependency}}} and it requires you to install either \href{https://github.com/sass/node-sass}{\texttt{ Node Sass}} or \href{https://github.com/sass/dart-sass}{\texttt{ Dart Sass}} on your own. This allows you to control the versions of all your dependencies, and to choose which Sass implementation to use.

\doxysubsection*{Examples}

Chain the sass-\/loader with the \href{https://github.com/webpack-contrib/css-loader}{\texttt{ css-\/loader}} and the \href{https://github.com/webpack-contrib/style-loader}{\texttt{ style-\/loader}} to immediately apply all styles to the DOM.


\begin{DoxyCode}{0}
\DoxyCodeLine{npm install style-\/loader css-\/loader -\/-\/save-\/dev}

\end{DoxyCode}



\begin{DoxyCode}{0}
\DoxyCodeLine{// webpack.config.js}
\DoxyCodeLine{module.exports = \{}
\DoxyCodeLine{    ...}
\DoxyCodeLine{    module: \{}
\DoxyCodeLine{        rules: [\{}
\DoxyCodeLine{            test: /\(\backslash\).scss\$/,}
\DoxyCodeLine{            use: [}
\DoxyCodeLine{                "{}style-\/loader"{}, // creates style nodes from JS strings}
\DoxyCodeLine{                "{}css-\/loader"{}, // translates CSS into CommonJS}
\DoxyCodeLine{                "{}sass-\/loader"{} // compiles Sass to CSS, using Node Sass by default}
\DoxyCodeLine{            ]}
\DoxyCodeLine{        \}]}
\DoxyCodeLine{    \}}
\DoxyCodeLine{\};}

\end{DoxyCode}


You can also pass options directly to \href{https://github.com/sass/node-sass}{\texttt{ Node Sass}} or \href{http://sass-lang.com/dart-sass}{\texttt{ Dart Sass}}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{// webpack.config.js}
\DoxyCodeLine{module.exports = \{}
\DoxyCodeLine{    ...}
\DoxyCodeLine{    module: \{}
\DoxyCodeLine{        rules: [\{}
\DoxyCodeLine{            test: /\(\backslash\).scss\$/,}
\DoxyCodeLine{            use: [\{}
\DoxyCodeLine{                loader: "{}style-\/loader"{}}
\DoxyCodeLine{            \}, \{}
\DoxyCodeLine{                loader: "{}css-\/loader"{}}
\DoxyCodeLine{            \}, \{}
\DoxyCodeLine{                loader: "{}sass-\/loader"{},}
\DoxyCodeLine{                options: \{}
\DoxyCodeLine{                    includePaths: ["{}absolute/path/a"{}, "{}absolute/path/b"{}]}
\DoxyCodeLine{                \}}
\DoxyCodeLine{            \}]}
\DoxyCodeLine{        \}]}
\DoxyCodeLine{    \}}
\DoxyCodeLine{\};}

\end{DoxyCode}


See \href{https://github.com/sass/node-sass/blob/master/README.md\#options}{\texttt{ the Node Sass documentation}} for all available Sass options.

By default the loader resolve the implementation based on your dependencies. Just add required implementation to {\ttfamily package.\+json} ({\ttfamily node-\/sass} or {\ttfamily sass} package) and install dependencies.

Example where the {\ttfamily sass-\/loader} loader uses the {\ttfamily sass} ({\ttfamily dart-\/sass}) implementation\+:

{\bfseries{package.\+json}}


\begin{DoxyCode}{0}
\DoxyCodeLine{\{}
\DoxyCodeLine{   "{}devDependencies"{}: \{}
\DoxyCodeLine{      "{}sass-\/loader"{}: "{}*"{},}
\DoxyCodeLine{      "{}sass"{}: "{}*"{}}
\DoxyCodeLine{   \}}
\DoxyCodeLine{\}}

\end{DoxyCode}


Example where the {\ttfamily sass-\/loader} loader uses the {\ttfamily node-\/sass} implementation\+:

{\bfseries{package.\+json}}


\begin{DoxyCode}{0}
\DoxyCodeLine{\{}
\DoxyCodeLine{   "{}devDependencies"{}: \{}
\DoxyCodeLine{      "{}sass-\/loader"{}: "{}*"{},}
\DoxyCodeLine{      "{}node-\/sass"{}: "{}*"{}}
\DoxyCodeLine{   \}}
\DoxyCodeLine{\}}

\end{DoxyCode}


Beware the situation when {\ttfamily node-\/sass} and {\ttfamily sass} was installed, by default the {\ttfamily sass-\/loader} prefers {\ttfamily node-\/sass}, to avoid this situation use the {\ttfamily implementation} option.

The special {\ttfamily implementation} option determines which implementation of Sass to use. It takes either a \href{https://github.com/sass/node-sass}{\texttt{ Node Sass}} or a \href{http://sass-lang.com/dart-sass}{\texttt{ Dart Sass}} module. For example, to use Dart Sass, you\textquotesingle{}d pass\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{// ...}
\DoxyCodeLine{    \{}
\DoxyCodeLine{        loader: "{}sass-\/loader"{},}
\DoxyCodeLine{        options: \{}
\DoxyCodeLine{            implementation: require("{}sass"{})}
\DoxyCodeLine{        \}}
\DoxyCodeLine{    \}}
\DoxyCodeLine{// ...}

\end{DoxyCode}


Note that when using Dart Sass, {\bfseries{synchronous compilation is twice as fast as asynchronous compilation}} by default, due to the overhead of asynchronous callbacks. To avoid this overhead, you can use the \href{https://www.npmjs.com/package/fibers}{\texttt{ {\ttfamily fibers}}} package to call asynchronous importers from the synchronous code path. To enable this, pass the {\ttfamily Fiber} class to the {\ttfamily fiber} option\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{// webpack.config.js}
\DoxyCodeLine{const Fiber = require('fibers');}
\DoxyCodeLine{}
\DoxyCodeLine{module.exports = \{}
\DoxyCodeLine{    ...}
\DoxyCodeLine{    module: \{}
\DoxyCodeLine{        rules: [\{}
\DoxyCodeLine{            test: /\(\backslash\).scss\$/,}
\DoxyCodeLine{            use: [\{}
\DoxyCodeLine{                loader: "{}style-\/loader"{}}
\DoxyCodeLine{            \}, \{}
\DoxyCodeLine{                loader: "{}css-\/loader"{}}
\DoxyCodeLine{            \}, \{}
\DoxyCodeLine{                loader: "{}sass-\/loader"{},}
\DoxyCodeLine{                options: \{}
\DoxyCodeLine{                    implementation: require("{}sass"{}),}
\DoxyCodeLine{                    fiber: Fiber}
\DoxyCodeLine{                \}}
\DoxyCodeLine{            \}]}
\DoxyCodeLine{        \}]}
\DoxyCodeLine{    \}}
\DoxyCodeLine{\};}

\end{DoxyCode}

\DoxyCodeLine{    plugins: [}
\DoxyCodeLine{        new MiniCssExtractPlugin(\{}

\DoxyCodeLine{        \})}
\DoxyCodeLine{    ]}
\DoxyCodeLine{\};}

\doxysubsection*{Usage}\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_sass_loader_README_autotoc_md22986}{}\doxysubsubsection{Imports}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_sass_loader_README_autotoc_md22986}
webpack provides an \href{https://webpack.js.org/concepts/module-resolution/}{\texttt{ advanced mechanism to resolve files}}. The sass-\/loader uses Sass\textquotesingle{}s custom importer feature to pass all queries to the webpack resolving engine. Thus you can import your Sass modules from {\ttfamily node\+\_\+modules}. Just prepend them with a {\ttfamily $\sim$} to tell webpack that this is not a relative import\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{@import "{}\string~bootstrap/dist/css/bootstrap"{};}

\end{DoxyCode}


It\textquotesingle{}s important to only prepend it with {\ttfamily $\sim$}, because {\ttfamily $\sim$/} resolves to the home directory. webpack needs to distinguish between {\ttfamily bootstrap} and {\ttfamily $\sim$bootstrap} because CSS and Sass files have no special syntax for importing relative files. Writing {\ttfamily @import \char`\"{}file\char`\"{}} is the same as {\ttfamily @import \char`\"{}./file\char`\"{};}\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_sass_loader_README_autotoc_md22987}{}\doxysubsubsection{Problems with $<$tt$>$url(...)$<$/tt$>$}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_sass_loader_README_autotoc_md22987}
Since Sass/\href{https://github.com/sass/libsass}{\texttt{ libsass}} does not provide \href{https://github.com/sass/libsass/issues/532}{\texttt{ url rewriting}}, all linked assets must be relative to the output.


\begin{DoxyItemize}
\item If you\textquotesingle{}re just generating CSS without passing it to the css-\/loader, it must be relative to your web root.
\item If you pass the generated CSS on to the css-\/loader, all urls must be relative to the entry-\/file (e.\+g. {\ttfamily main.\+scss}).
\end{DoxyItemize}

More likely you will be disrupted by this second issue. It is natural to expect relative references to be resolved against the {\ttfamily .scss} file in which they are specified (like in regular {\ttfamily .css} files). Thankfully there are a two solutions to this problem\+:


\begin{DoxyItemize}
\item Add the missing url rewriting using the \href{https://github.com/bholloway/resolve-url-loader}{\texttt{ resolve-\/url-\/loader}}. Place it before the sass-\/loader in the loader chain.
\item Library authors usually provide a variable to modify the asset path. \href{https://github.com/twbs/bootstrap-sass}{\texttt{ bootstrap-\/sass}} for example has an {\ttfamily \$icon-\/font-\/path}. Check out \href{https://github.com/webpack-contrib/sass-loader/tree/master/test/bootstrapSass}{\texttt{ this working bootstrap example}}.
\end{DoxyItemize}\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_sass_loader_README_autotoc_md22988}{}\doxysubsubsection{Extracting style sheets}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_sass_loader_README_autotoc_md22988}
Bundling CSS with webpack has some nice advantages like referencing images and fonts with hashed urls or \href{https://webpack.js.org/concepts/hot-module-replacement/}{\texttt{ hot module replacement}} in development. In production, on the other hand, it\textquotesingle{}s not a good idea to apply your style sheets depending on JS execution. Rendering may be delayed or even a \href{https://en.wikipedia.org/wiki/Flash_of_unstyled_content}{\texttt{ FOUC}} might be visible. Thus it\textquotesingle{}s often still better to have them as separate files in your final production build.

There are two possibilities to extract a style sheet from the bundle\+:


\begin{DoxyItemize}
\item \href{https://github.com/peerigon/extract-loader}{\texttt{ extract-\/loader}} (simpler, but specialized on the css-\/loader\textquotesingle{}s output)
\item \href{https://github.com/webpack-contrib/mini-css-extract-plugin}{\texttt{ mini-\/css-\/extract-\/plugin}} (use this, when using webpack 4 configuration. Works in all use-\/cases)
\end{DoxyItemize}\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_sass_loader_README_autotoc_md22989}{}\doxysubsubsection{Source maps}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_sass_loader_README_autotoc_md22989}
To enable CSS source maps, you\textquotesingle{}ll need to pass the {\ttfamily source\+Map} option to the sass-\/loader {\itshape and} the css-\/loader. Your {\ttfamily webpack.\+config.\+js} should look like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{module.exports = \{}
\DoxyCodeLine{    ...}
\DoxyCodeLine{    devtool: "{}source-\/map"{}, // any "{}source-\/map"{}-\/like devtool is possible}
\DoxyCodeLine{    module: \{}
\DoxyCodeLine{        rules: [\{}
\DoxyCodeLine{            test: /\(\backslash\).scss\$/,}
\DoxyCodeLine{            use: [\{}
\DoxyCodeLine{                loader: "{}style-\/loader"{}, options: \{}
\DoxyCodeLine{                    sourceMap: true}
\DoxyCodeLine{                \}}
\DoxyCodeLine{            \}, \{}
\DoxyCodeLine{                loader: "{}css-\/loader"{}, options: \{}
\DoxyCodeLine{                    sourceMap: true}
\DoxyCodeLine{                \}}
\DoxyCodeLine{            \}, \{}
\DoxyCodeLine{                loader: "{}sass-\/loader"{}, options: \{}
\DoxyCodeLine{                    sourceMap: true}
\DoxyCodeLine{                \}}
\DoxyCodeLine{            \}]}
\DoxyCodeLine{        \}]}
\DoxyCodeLine{    \}}
\DoxyCodeLine{\};}

\end{DoxyCode}


If you want to edit the original Sass files inside Chrome, \href{https://medium.com/@toolmantim/getting-started-with-css-sourcemaps-and-in-browser-sass-editing-b4daab987fb0}{\texttt{ there\textquotesingle{}s a good blog post}}. Checkout \href{https://github.com/webpack-contrib/sass-loader/tree/master/test}{\texttt{ test/source\+Map}} for a running example.\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_sass_loader_README_autotoc_md22990}{}\doxysubsubsection{Environment variables}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_sass_loader_README_autotoc_md22990}
If you want to prepend Sass code before the actual entry file, you can set the {\ttfamily data} option. In this case, the sass-\/loader will not override the {\ttfamily data} option but just append the entry\textquotesingle{}s content. This is especially useful when some of your Sass variables depend on the environment\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\{}
\DoxyCodeLine{    loader: "{}sass-\/loader"{},}
\DoxyCodeLine{    options: \{}
\DoxyCodeLine{        data: "{}\$env: "{} + process.env.NODE\_ENV + "{};"{}}
\DoxyCodeLine{    \}}
\DoxyCodeLine{\}}

\end{DoxyCode}


The {\ttfamily data} option supports {\ttfamily Function} notation\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\{}
\DoxyCodeLine{    loader: "{}sass-\/loader"{},}
\DoxyCodeLine{    options: \{}
\DoxyCodeLine{        data: (loaderContext) => \{}
\DoxyCodeLine{          // More information about avalaible options https://webpack.js.org/api/loaders/}
\DoxyCodeLine{          const \{ resourcePath, rootContext \} = loaderContext;}
\DoxyCodeLine{          const relativePath = path.relative(rootContext,resourcePath);}
\DoxyCodeLine{          }
\DoxyCodeLine{          if (relativePath === "{}styles/foo.scss"{}) \{}
\DoxyCodeLine{             return "{}\$value: 100px;"{}}
\DoxyCodeLine{          \}}
\DoxyCodeLine{          }
\DoxyCodeLine{          return "{}\$value: 200px;"{}}
\DoxyCodeLine{        \}}
\DoxyCodeLine{    \}}
\DoxyCodeLine{\}}

\end{DoxyCode}


{\bfseries{Please note\+:}} Since you\textquotesingle{}re injecting code, this will break the source mappings in your entry file. Often there\textquotesingle{}s a simpler solution than this, like multiple Sass entry files.
