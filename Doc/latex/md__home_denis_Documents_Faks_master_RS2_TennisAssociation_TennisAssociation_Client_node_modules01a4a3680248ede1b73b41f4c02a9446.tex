Parses {\ttfamily sri} into an {\ttfamily Integrity} data structure. {\ttfamily sri} can be an integrity string, an {\ttfamily Hash}-\/like with {\ttfamily digest} and {\ttfamily algorithm} fields and an optional {\ttfamily options} field, or an {\ttfamily Integrity}-\/like object. The resulting object will be an {\ttfamily Integrity} instance that has this shape\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\{}
\DoxyCodeLine{  'sha1': [\{algorithm: 'sha1', digest: 'deadbeef', options: []\}],}
\DoxyCodeLine{  'sha512': [}
\DoxyCodeLine{    \{algorithm: 'sha512', digest: 'c0ffee', options: []\},}
\DoxyCodeLine{    \{algorithm: 'sha512', digest: 'bad1dea', options: ['foo']\}}
\DoxyCodeLine{  ],}
\DoxyCodeLine{\}}

\end{DoxyCode}


If {\ttfamily opts.\+single} is truthy, a single {\ttfamily Hash} object will be returned. That is, a single object that looks like {\ttfamily \{algorithm, digest, options\}}, as opposed to a larger object with multiple of these.

If {\ttfamily opts.\+strict} is truthy, the resulting object will be filtered such that it strictly follows the Subresource Integrity spec, throwing away any entries with any invalid components. This also means a restricted set of algorithms will be used -- the spec limits them to {\ttfamily sha256}, {\ttfamily sha384}, and {\ttfamily sha512}.

Strict mode is recommended if the integrity strings are intended for use in browsers, or in other situations where strict adherence to the spec is needed.\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules01a4a3680248ede1b73b41f4c02a9446_autotoc_md674}{}\doxyparagraph{Example}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules01a4a3680248ede1b73b41f4c02a9446_autotoc_md674}
