smart-\/buffer is a light Buffer wrapper that takes away the need to keep track of what position to read and write data to and from the underlying Buffer. It also adds null terminating string operations and {\bfseries{grows}} as you add more data.

\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_smart_buffer_docs_README_v3_autotoc_md12897}{}\doxysubsection{What it\textquotesingle{}s useful for\+:}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_smart_buffer_docs_README_v3_autotoc_md12897}
I created smart-\/buffer because I wanted to simplify the process of using Buffer for building and reading network packets to send over a socket. Rather than having to keep track of which position I need to write a UInt16 to after adding a string of variable length, I simply don\textquotesingle{}t have to.

Key Features\+:
\begin{DoxyItemize}
\item Proxies all of the Buffer write and read functions.
\item Keeps track of read and write positions for you.
\item Grows the internal Buffer as you add data to it.
\item Useful string operations. (Null terminating strings)
\item Allows for inserting values at specific points in the internal Buffer.
\item Built in Type\+Script
\item Type Definitions Provided
\end{DoxyItemize}

Requirements\+:
\begin{DoxyItemize}
\item Node v4.\+0+ is supported at this time. (Versions prior to 2.\+0 will work on node 0.\+10)
