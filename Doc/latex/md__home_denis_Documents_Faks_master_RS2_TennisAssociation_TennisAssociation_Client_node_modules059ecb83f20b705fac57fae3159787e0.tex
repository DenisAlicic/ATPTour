Version 5 and 6 of the npm CLI make a request to the registry endpoint at either the \char`\"{}\+Full Audit\char`\"{} endpoint at {\ttfamily /-\//npm/v1/security/audits} or the \char`\"{}\+Quick Audit\char`\"{} endpoint at {\ttfamily /-\//npm/v1/security/audits/quick}. The Full Audit endpoint calculates remediations necessary to correct problems based on the shape of the tree.

As of npm v7, the logic of how the cli manages trees is dramatically rearchitected, rendering much of the remediations no longer valid. Thus, it {\itshape only} fetches the advisory data from the Quick Audit endpoint, and uses \href{http://npm.im/@npmcli/arborist}{\texttt{ {\ttfamily @npmcli/arborist}}} to calculate required remediations and affected nodes in the dependency graph. This data is serialized and provided as an {\ttfamily \char`\"{}audit\+Report\+Version\char`\"{}\+: 2} object.

Version 2 of this module expects to recieve an instance (or serialized JSON version of) the {\ttfamily Audit\+Report} class from Arborist, which is returned by {\ttfamily arborist.\+audit()} and stored on the instance as {\ttfamily arborist.\+audit\+Report}.
