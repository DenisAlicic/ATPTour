{\ttfamily default\+Severity} defined in a {\ttfamily tslint.\+json} file will now override the {\ttfamily default\+Severity} value defined in any configurations you are extending. This means that any rules specified in the base configurations can now take on a new {\ttfamily default\+Severity} if you so choose. If you extend multiple configuration files, the {\ttfamily default\+Severity} defined in the last one wins.

In practice, this allows users to, for example, more easily use the built-\/in TSLint configurations ({\ttfamily tslint\+:recommended}, {\ttfamily tslint\+:latest}, {\ttfamily tslint\+:all}) and treat all errors as warnings instead of errors.
