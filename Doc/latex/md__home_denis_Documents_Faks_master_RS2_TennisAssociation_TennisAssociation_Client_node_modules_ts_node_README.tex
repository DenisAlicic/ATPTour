\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_ts_node_README_autotoc_md21840}{}\doxysection{How It Works}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_ts_node_README_autotoc_md21840}
{\bfseries{Type\+Script Node}} works by registering the Type\+Script compiler for {\ttfamily .tsx?} and {\ttfamily .jsx?} (when {\ttfamily allow\+Js == true}) extensions. When node.\+js has an extension registered (via {\ttfamily require.\+extensions}), it will use the extension internally for module resolution. When an extension is unknown to node.\+js, it handles the file as {\ttfamily .js} (Java\+Script). By default, {\bfseries{Type\+Script Node}} avoids compiling files in {\ttfamily /node\+\_\+modules/} for three reasons\+:


\begin{DoxyEnumerate}
\item Modules should always be published in a format node.\+js can consume
\item Transpiling the entire dependency tree will make your project slower
\item Differing behaviours between Type\+Script and node.\+js (e.\+g. ES2015 modules) can result in a project that works until you decide to support a feature natively from node.\+js
\end{DoxyEnumerate}

{\bfseries{P.\+S.}} This means if you don\textquotesingle{}t register an extension, it is compiled as Java\+Script. When {\ttfamily ts-\/node} is used with {\ttfamily allow\+Js}, Java\+Script files are transpiled using the Type\+Script compiler.\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_ts_node_README_autotoc_md21841}{}\doxysection{Loading $<$tt$>$tsconfig.\+json$<$/tt$>$}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_ts_node_README_autotoc_md21841}
{\bfseries{Typescript Node}} loads {\ttfamily tsconfig.\+json} automatically. Use {\ttfamily -\/-\/skip-\/project} to the loading {\ttfamily tsconfig.\+json}.

{\bfseries{Type\+Script Node}} does {\itshape not} use {\ttfamily files}, {\ttfamily include} or {\ttfamily exclude}, by default. This is because a large majority projects do not use all of the files in a project directory (e.\+g. {\ttfamily Gulpfile.\+ts}, runtime vs tests) and parsing every file for types slows startup time. Instead, {\ttfamily ts-\/node} starts with the script file (e.\+g. {\ttfamily ts-\/node index.\+ts}) and Type\+Script resolves dependencies based on imports and references.
