\item Public Beta / Release Candidate -\/ Feedback welcome For a guide to breaking changes see \href{http://lesscss.org/usage/\#v2-upgrade-guide}{\texttt{ the v2 upgrade guide}}
\item no longer including old versions of less in the repo or npm
\item not including test less and gradle files in npm
\item colours now output in the format they are added, so yellow will output yellow, not its hex counterpart
\item better parsing -\/ better comment support and comments in brackets can now contain comments including quotes.
\item Removal of dependency on clean-\/css -\/ install less-\/plugin-\/clean-\/css and use --clean-\/css to reference plugin
\item Environment Support -\/ less is now separate from its node and browser environment implementations and adding support for another javascript environment should be straight forward.
\item Plugin Support -\/ it is now straight forward to add AST manipulations (see less-\/plugin-\/inline-\/images), file managers (see less-\/plugin-\/npm-\/import) and post processors (see less-\/plugin-\/clean-\/css and less-\/plugin-\/autoprefix).
\item We now recommend using less.\+render and using the parser directly is not in the same way as in v2. It is possible but it would require changes and we do not guarantee it will not be broken in minor version releases.
\item In the browser, less.\+page\+Load\+Finished will be a promise, resolved when less has finished its initial processing. less.\+refresh and less.\+modify\+Vars also return promises.
\item In the browser, as before less is used as options, however this is now copied to less.\+options if you need to access after less has run
\item In the browser, the cache can be overwritten by setting less.\+cache before less loads. After load less.\+cache will be the default implementation.
\item less.\+js now uses browserify to generate its browser side component
\item default values for the sourcemap options have been re-\/done and improved to hopefully mean creating sourcemaps is easier
\item Many smaller bugfixes and API changes. Please let us know if something you relied on has disappeared or an area should be better documented.
