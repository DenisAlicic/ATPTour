This is a \href{https://webpack.js.org/loaders/}{\texttt{ Webpack loader}} that uses \href{https://istanbul.js.org/}{\texttt{ Istanbul}} to add code coverage instrumentation to your code. This is especially useful for in-\/browser testing, in a tool like \href{https://karma-runner.github.io/3.0/index.html}{\texttt{ Karma}}, where you can use \href{https://github.com/mattlewis92/karma-coverage-istanbul-reporter}{\texttt{ karma-\/coverage-\/istanbul-\/reporter}} to output code coverage data in a variety of formats that are supported by services like \href{https://coveralls.io/}{\texttt{ Coveralls}} or \href{https://www.codacy.com/}{\texttt{ Codacy}}.

Coverage Istanbul Loader is based on \href{https://github.com/webpack-contrib/istanbul-instrumenter-loader}{\texttt{ Istanbul Instrumenter Loader}}, but uses a newer version of the \href{https://github.com/istanbuljs/istanbuljs/tree/master/packages/istanbul-lib-instrument}{\texttt{ Istanbul API}} and has better support for source maps.\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modulesefb7bfdd7a800bcfcca4190a2f6cf6d2_autotoc_md18875}{}\doxysection{Usage}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modulesefb7bfdd7a800bcfcca4190a2f6cf6d2_autotoc_md18875}
