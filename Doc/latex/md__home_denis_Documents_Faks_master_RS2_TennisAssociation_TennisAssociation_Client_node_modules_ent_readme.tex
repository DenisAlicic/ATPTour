Escape unsafe characters in {\ttfamily str} with html entities.

By default, entities are encoded with numeric decimal codes.

If {\ttfamily opts.\+numeric} is false or {\ttfamily opts.\+named} is true, encoding will used named codes like {\ttfamily {$\pi$}}.

If {\ttfamily opts.\+special} is set to an Object, the key names will be forced to be encoded (defaults to forcing\+: `$<$$>$'"{}\&\`{}). For example\+:
