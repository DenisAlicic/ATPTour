An implementation of window.\+fetch in Node.\+js using Minipass streams

This is a fork (or more precisely, a reimplementation) of \href{http://npm.im/node-fetch}{\texttt{ node-\/fetch}}. All streams have been replaced with \href{http://npm.im/minipass}{\texttt{ minipass streams}}.

The goal of this module is to stay in sync with the API presented by {\ttfamily node-\/fetch}, with the exception of the streaming interface provided.\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modulesf7e3d6f9332dd2a5d7cbd2aac0180a6e_autotoc_md13828}{}\doxysection{Why}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modulesf7e3d6f9332dd2a5d7cbd2aac0180a6e_autotoc_md13828}
Minipass streams are faster and more deterministic in their timing contract than node-\/core streams, making them a better fit for many server-\/side use cases.\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modulesf7e3d6f9332dd2a5d7cbd2aac0180a6e_autotoc_md13829}{}\doxysection{API}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modulesf7e3d6f9332dd2a5d7cbd2aac0180a6e_autotoc_md13829}
See \href{http://npm.im/node-fetch}{\texttt{ node-\/fetch}}

Differences from {\ttfamily node-\/fetch} (and, by extension, from the What\+WG Fetch specification)\+:


\begin{DoxyItemize}
\item Returns \href{http://npm.im/minipass}{\texttt{ minipass}} streams instead of node-\/core streams.
\item Supports the full set of \href{https://nodejs.org/api/https.html\#https_https_request_options_callback}{\texttt{ TLS Options that may be provided to {\ttfamily https.\+request()}}} when making {\ttfamily https} requests. 
\end{DoxyItemize}
