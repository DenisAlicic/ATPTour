Cosmiconfig searches for and loads configuration for your program.

It features smart defaults based on conventional expectations in the Java\+Script ecosystem. But it\textquotesingle{}s also flexible enough to search wherever you\textquotesingle{}d like to search, and load whatever you\textquotesingle{}d like to load.

By default, Cosmiconfig will start where you tell it to start and search up the directory tree for the following\+:


\begin{DoxyItemize}
\item a {\ttfamily package.\+json} property
\item a JSON or YAML, extensionless \char`\"{}rc file\char`\"{}
\item an \char`\"{}rc file\char`\"{} with the extensions {\ttfamily .json}, {\ttfamily .yaml}, {\ttfamily .yml}, or {\ttfamily .js}.
\item a {\ttfamily .config.\+js} Common\+JS module
\end{DoxyItemize}

For example, if your module\textquotesingle{}s name is \char`\"{}soursocks\char`\"{}, cosmiconfig will search up the directory tree for configuration in the following places\+:


\begin{DoxyItemize}
\item a {\ttfamily soursocks} property in {\ttfamily package.\+json}
\item a {\ttfamily .soursocksrc} file in JSON or YAML format
\item a {\ttfamily .soursocksrc.\+json} file
\item a {\ttfamily .soursocksrc.\+yaml}, {\ttfamily .soursocksrc.\+yml}, or {\ttfamily .soursocksrc.\+js} file
\item a {\ttfamily soursocks.\+config.\+js} file exporting a JS object
\end{DoxyItemize}

Cosmiconfig continues to search up the directory tree, checking each of these places in each directory, until it finds some acceptable configuration (or hits the home directory).

Searches for a configuration file. Returns a Promise that resolves with a \href{\#result}{\texttt{ result}} or with {\ttfamily null}, if no configuration file is found.

You can do the same thing synchronously with \href{\#explorersearchsync}{\texttt{ {\ttfamily search\+Sync()}}}.

Let\textquotesingle{}s say your module name is {\ttfamily goldengrahams} so you initialized with `const explorer = cosmiconfig(\textquotesingle{}goldengrahams');{\ttfamily . Here\textquotesingle{}s how your default \mbox{[}}search()\`{}\mbox{]} will work\+:


\begin{DoxyItemize}
\item Starting from {\ttfamily \mbox{\hyperlink{namespaceupdate-gyp_a353f325a85a850b0456a7cbaff8fc43b}{process.\+cwd()}}} (or some other directory defined by the {\ttfamily search\+From} argument to \href{\#explorersearch}{\texttt{ {\ttfamily search()}}}), look for configuration objects in the following places\+:
\begin{DoxyEnumerate}
\item A {\ttfamily goldengrahams} property in a {\ttfamily package.\+json} file.
\item A {\ttfamily .goldengrahamsrc} file with JSON or YAML syntax.
\item A {\ttfamily .goldengrahamsrc.\+json} file.
\item A {\ttfamily .goldengrahamsrc.\+yaml}, {\ttfamily .goldengrahamsrc.\+yml}, or {\ttfamily .goldengrahamsrc.\+js} file.
\item A {\ttfamily goldengrahams.\+config.\+js} JS file exporting the object.
\end{DoxyEnumerate}
\item If none of those searches reveal a configuration object, move up one directory level and try again. So the search continues in {\ttfamily ./}, {\ttfamily ../}, {\ttfamily ../../}, {\ttfamily ../../../}, etc., checking the same places in each directory.
\item Continue searching until arriving at your home directory (or some other directory defined by the cosmiconfig option \href{\#stopdir}{\texttt{ {\ttfamily stop\+Dir}}}).
\item If at any point a parseable configuration is found, the \href{\#explorersearch}{\texttt{ {\ttfamily search()}}} Promise resolves with its \href{\#result}{\texttt{ result}} (or, with \href{\#explorersearchsync}{\texttt{ {\ttfamily search\+Sync()}}}, the \href{\#result}{\texttt{ result}} is returned).
\item If no configuration object is found, the \href{\#explorersearch}{\texttt{ {\ttfamily search()}}} Promise resolves with {\ttfamily null} (or, with \href{\#explorersearchsync}{\texttt{ {\ttfamily search\+Sync()}}}, {\ttfamily null} is returned).
\item If a configuration object is found {\itshape but is malformed} (causing a parsing error), the \href{\#explorersearch}{\texttt{ {\ttfamily search()}}} Promise rejects with that error (so you should {\ttfamily .catch()} it). (Or, with \href{\#explorersearchsync}{\texttt{ {\ttfamily search\+Sync()}}}, the error is thrown.)
\end{DoxyItemize}

{\bfseries{If you provide a {\ttfamily loaders} object, your object will be {\itshape merged} with the defaults.}} So you can override one or two without having to override them all.

{\bfseries{Keys in {\ttfamily loaders}}} are extensions (starting with a period), or {\ttfamily no\+Ext} to specify the loader for files {\itshape without} extensions, like {\ttfamily .soursocksrc}.

{\bfseries{Values in {\ttfamily loaders}}} are either a loader function (described below) or an object with {\ttfamily sync} and/or {\ttfamily async} properties, whose values are loader functions.

{\bfseries{The most common use case for custom loaders value is to load extensionless {\ttfamily rc} files as strict JSON}}, instead of JSON {\itshape or} YAML (the default). To accomplish that, provide the following {\ttfamily loaders} value\+:

\begin{DoxyItemize}
\item Allow configuration syntaxes that aren\textquotesingle{}t handled by Cosmiconfig\textquotesingle{}s defaults, like JSON5, INI, or XML.

{\bfseries{Custom loader functions}} have the following signature\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{// Sync}
\DoxyCodeLine{(filepath: string, content: string) => Object | null}
\DoxyCodeLine{}
\DoxyCodeLine{// Async}
\DoxyCodeLine{(filepath: string, content: string) => Object | null | Promise<Object | null>}

Cosmiconfig reads the file when it checks whether the file exists, so it will provide you with both the file\textquotesingle{}s path and its content. Do whatever you need to, and return either a configuration object or {\ttfamily null} (or, for async-\/only loaders, a Promise that resolves with one of those). {\ttfamily null} indicates that no real configuration was found and the search should continue.

It\textquotesingle{}s easiest if you make your custom loader function synchronous. Then it can be used regardless of whether you end up calling \href{\#explorersearch}{\texttt{ {\ttfamily search()}}} or \href{\#explorersearchsync}{\texttt{ {\ttfamily search\+Sync()}}}, \href{\#explorerload}{\texttt{ {\ttfamily load()}}} or \href{\#explorerloadsync}{\texttt{ {\ttfamily load\+Sync()}}}. If you want or need to provide an async-\/only loader, you can do so by making the value of {\ttfamily loaders} an object with an {\ttfamily async} property whose value is the async loader. You can also add a {\ttfamily sync} property to designate a sync loader, if you want to use both async and sync search and load functions.

A few things to note\+:


\begin{DoxyItemize}
\item If you use a custom loader, be aware of whether it\textquotesingle{}s sync or async and how that aligned with your usage of sync or async search and load functions.
\item {\bfseries{Special JS syntax can also be handled by using a {\ttfamily require} hook}}, because {\ttfamily cosmiconfig.\+load\+Js} just uses {\ttfamily require}. Whether you use custom loaders or a {\ttfamily require} hook is up to you.
\end{DoxyItemize}
