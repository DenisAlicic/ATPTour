\href{https://npmjs.com/package/mini-css-extract-plugin}{\texttt{ }} \href{https://david-dm.org/webpack-contrib/mini-css-extract-plugin}{\texttt{ }} \href{https://dev.azure.com/webpack-contrib/mini-css-extract-plugin/_build/latest?definitionId=6&branchName=master}{\texttt{ }} \href{https://codecov.io/gh/webpack-contrib/mini-css-extract-plugin}{\texttt{ }} \href{https://gitter.im/webpack/webpack}{\texttt{ }}

This plugin extracts CSS into separate files. It creates a CSS file per JS file which contains CSS. It supports On-\/\+Demand-\/\+Loading of CSS and Source\+Maps.

It builds on top of a new webpack v4 feature (module types) and requires webpack 4 to work.

Compared to the extract-\/text-\/webpack-\/plugin\+:


\begin{DoxyItemize}
\item Async loading
\item No duplicate compilation (performance)
\item Easier to use
\item Specific to CSS
\end{DoxyItemize}
