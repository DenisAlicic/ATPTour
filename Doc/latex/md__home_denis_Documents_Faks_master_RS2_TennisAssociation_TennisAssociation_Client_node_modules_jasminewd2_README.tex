Adapter for Jasmine-\/to-\/\+Web\+Driver\+JS. Used by \href{http://www.github.com/angular/protractor}{\texttt{ Protractor}}.

{\bfseries{Important\+:}} There are two active branches of jasminewd.


\begin{DoxyItemize}
\item \href{https://github.com/angular/jasminewd/tree/jasminewd1}{\texttt{ jasminewd1}} is an adapter for Jasmine 1.\+3, and uses the package minijasminenode. It is published to npm as {\ttfamily jasminewd}.
\item \href{https://github.com/angular/jasminewd/tree/jasminewd2}{\texttt{ jasminewd2}} is an adapter for Jasmine 2.\+x, and uses the package jasmine. It is published to npm as {\ttfamily jasminewd2}.

\begin{DoxyItemize}
\item Automatically makes tests asynchronously wait until the Web\+Driver\+JS control flow is empty.
\item If a {\ttfamily done} function is passed to the test, waits for both the control flow and until done is called.
\item If a test returns a promise, waits for both the control flow and the promise to resolve.
\item Enhances {\ttfamily expect} so that it automatically unwraps promises before performing the assertion.

For the typings related to the changes in the global jasmine variables (e.\+g. allowing {\ttfamily it()} blocks to return a promise), we publish the package {\ttfamily @types/jasminewd2}. If you are writing tests using jasminewd (including Protractor tests), be sure to include {\ttfamily @types/jasminewd2} in your {\ttfamily dev\+Dependencies}, as these global type modifications are {\itshape {\bfseries{not}}} bundled with the {\ttfamily jasminewd2} npm module.
