There are a couple cases where Protractor launches downloaded binaries (from webdriver-\/manager) without specifying which version to use\+:


\begin{DoxyItemize}
\item local driver providers -\/ when no {\ttfamily selenium\+Address}, {\ttfamily direct\+Connect}, saucelabs or browser stack configuration is set
\item direct connect -\/ when setting {\ttfamily direct\+Connect} to true
\end{DoxyItemize}

Protractor knows which downloaded binaries to use based on the {\ttfamily update-\/config.\+json}. The {\ttfamily update-\/config.\+json} is written during the {\ttfamily update} command and provides the path to the current downloaded version and a list of all the binaries previously downloaded.

During the launch of either local driver provider or direct connect, Protractor will launch the last downloaded binary.
