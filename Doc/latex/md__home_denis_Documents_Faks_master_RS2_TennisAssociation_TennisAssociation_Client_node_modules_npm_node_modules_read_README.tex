The callback gets called with either the user input, or the default specified, or an error, as {\ttfamily callback(error, result, is\+Default)} node style.\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_npm_node_modules_read_README_autotoc_md16004}{}\doxysection{OPTIONS}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_npm_node_modules_read_README_autotoc_md16004}
Every option is optional.


\begin{DoxyItemize}
\item {\ttfamily prompt} What to write to stdout before reading input.
\item {\ttfamily silent} Don\textquotesingle{}t echo the output as the user types it.
\item {\ttfamily replace} Replace silenced characters with the supplied character value.
\item {\ttfamily timeout} Number of ms to wait for user input before giving up.
\item {\ttfamily default} The default value if the user enters nothing.
\item {\ttfamily edit} Allow the user to edit the default value.
\item {\ttfamily terminal} Treat the output as a TTY, whether it is or not.
\item {\ttfamily input} Readable stream to get input data from. (default {\ttfamily process.\+stdin})
\item {\ttfamily output} Writeable stream to write prompts to. (default\+: {\ttfamily process.\+stdout})
\end{DoxyItemize}

This module works sort of with node 0.\+6. It does not work with node versions less than 0.\+6. It is best on node 0.\+8.

On node version 0.\+6, it will remove all listeners on the input stream\textquotesingle{}s {\ttfamily data} and {\ttfamily keypress} events, because the readline module did not fully clean up after itself in that version of node, and did not make it possible to clean up after it in a way that has no potential for side effects.
