Overview when you call {\ttfamily update}\+:


\begin{DoxyItemize}
\item The output directory is created if it does not exist ({\ttfamily out\+\_\+dir})
\item If this is the first time downloading, it will make web requests to selenium-\/release.\+storage.\+googleapis.\+com, chromedriver.\+storage.\+googleapis.\+com, and api.\+github.\+com. The responses will be stored in the {\ttfamily out\+\_\+dir} as a cache.
\item If this is not the first time downloading, it will look at the cached files in {\ttfamily out\+\_\+dir} and if the file is older than an hour, it will make a request for new files. Note\+: api.\+github.\+com is rate limited by the ip address making the request. This is an issue for common CI systems like Travis or Circle and have an issue opened for this.
\item The response will then be parsed for the url to get the latest version if no version is provided via command line. The latest version will be based on what is available for your OS type and architecture.
\item Next we download the file from the url. If the file already exists (based on content length), we cancel the download.
\item We uncompress zip and tar files and for linux and mac, we set the permissions. Currently, we always uncompress the files even if the uncompressed files already exist.
\item Finally we write the file locations to the {\ttfamily update-\/config.\+json} which is also located in the {\ttfamily out\+\_\+dir}.
