There are one directive for all chart types\+: {\ttfamily base\+Chart}, and there are 8 types of charts\+: {\ttfamily line}, {\ttfamily bar}, {\ttfamily radar}, {\ttfamily pie}, {\ttfamily polar\+Area}, {\ttfamily doughnut}, {\ttfamily bubble} and {\ttfamily scatter}.\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_ng2_charts_README_autotoc_md7531}{}\doxysubsubsection{Properties}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_ng2_charts_README_autotoc_md7531}
{\bfseries{Note}}\+: For more information about possible options please refer to original \href{http://www.chartjs.org/docs}{\texttt{ chart.\+js}} documentation


\begin{DoxyItemize}
\item {\ttfamily data} ({\ttfamily Single\+Or\+Multi\+Data\+Set}) -\/ set of points of the chart, it should be {\ttfamily Multi\+Data\+Set} only for {\ttfamily line}, {\ttfamily bar}, {\ttfamily radar} and {\ttfamily doughnut}, otherwise {\ttfamily Single\+Data\+Set}
\item {\ttfamily datasets} ({\ttfamily \{ data\+: Single\+Data\+Set, label\+: string \}\mbox{[}\mbox{]}}) -\/ {\ttfamily data} see about, the {\ttfamily label} for the dataset which appears in the legend and tooltips
\item {\ttfamily labels} ({\ttfamily Label\mbox{[}\mbox{]}}) -\/ x axis labels. It\textquotesingle{}s necessary for charts\+: {\ttfamily line}, {\ttfamily bar} and {\ttfamily radar}. And just labels (on hover) for charts\+: {\ttfamily polar\+Area}, {\ttfamily pie} and {\ttfamily doughnut}. {\ttfamily Label} is either a single {\ttfamily string}, or it may be a {\ttfamily string\mbox{[}\mbox{]}} representing a multi-\/line label where each array element is on a new line.
\item {\ttfamily chart\+Type} ({\ttfamily Chart\+Type}) -\/ indicates the type of charts, it can be\+: {\ttfamily line}, {\ttfamily bar}, {\ttfamily radar}, {\ttfamily pie}, {\ttfamily polar\+Area}, {\ttfamily doughnut}
\item {\ttfamily options} ({\ttfamily Chart\+Options}) -\/ chart options (as from \href{http://www.chartjs.org/docs/}{\texttt{ Chart.\+js documentation}})
\item {\ttfamily colors} ({\ttfamily Color\mbox{[}\mbox{]}}) -\/ data colors, will use default and$\vert$or random colors if not specified (see below)
\item {\ttfamily legend}\+: ({\ttfamily boolean = false}) -\/ if true show legend below the chart, otherwise not be shown

The {\ttfamily overrides} object has the same type as the chart options object {\ttfamily Chart\+Options}, and wherever a simple field is encountered it replaces the matching field in the {\ttfamily options} object.
