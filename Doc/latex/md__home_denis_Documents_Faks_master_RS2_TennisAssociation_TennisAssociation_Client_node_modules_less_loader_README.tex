Starting with {\ttfamily less-\/loader} 4, you can now choose between Less\textquotesingle{} builtin resolver and webpack\textquotesingle{}s resolver. By default, webpack\textquotesingle{}s resolver is used.\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_less_loader_README_autotoc_md21108}{}\doxyparagraph{webpack resolver}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_less_loader_README_autotoc_md21108}
webpack provides an \href{https://webpack.js.org/configuration/resolve/}{\texttt{ advanced mechanism to resolve files}}. The {\ttfamily less-\/loader} applies a Less plugin that passes all queries to the webpack resolver. Thus you can import your Less modules from {\ttfamily node\+\_\+modules}. Just prepend them with a {\ttfamily $\sim$} which tells webpack to look up the \href{https://webpack.js.org/configuration/resolve/\#resolve-modules}{\texttt{ {\ttfamily modules}}}.


\begin{DoxyCode}{0}
\DoxyCodeLine{@import '\string~bootstrap/less/bootstrap';}

\end{DoxyCode}


It\textquotesingle{}s important to only prepend it with {\ttfamily $\sim$}, because {\ttfamily $\sim$/} resolves to the home-\/directory. webpack needs to distinguish between {\ttfamily bootstrap} and {\ttfamily $\sim$bootstrap}, because CSS and Less files have no special syntax for importing relative files. Writing {\ttfamily @import \char`\"{}file\char`\"{}} is the same as {\ttfamily @import \char`\"{}./file\char`\"{};}

Bundling CSS with webpack has some nice advantages like referencing images and fonts with hashed urls or \href{https://webpack.js.org/concepts/hot-module-replacement/}{\texttt{ hot module replacement}} in development. In production, on the other hand, it\textquotesingle{}s not a good idea to apply your style sheets depending on JS execution. Rendering may be delayed or even a \href{https://en.wikipedia.org/wiki/Flash_of_unstyled_content}{\texttt{ FOUC}} might be visible. Thus it\textquotesingle{}s often still better to have them as separate files in your final production build.

There are two possibilities to extract a style sheet from the bundle\+:


\begin{DoxyItemize}
\item \href{https://github.com/peerigon/extract-loader}{\texttt{ {\ttfamily extract-\/loader}}} (simpler, but specialized on the css-\/loader\textquotesingle{}s output)
\item \href{https://github.com/webpack-contrib/mini-css-extract-plugin}{\texttt{ Mini\+Css\+Extract\+Plugin}} (more complex, but works in all use-\/cases)
