Autoprefixer uses \href{https://github.com/ai/browserslist}{\texttt{ Browserslist}}, so you can specify the browsers you want to target in your project with queries like {\ttfamily $>$ 5\%} (see \href{https://github.com/browserslist/browserslist\#best-practices}{\texttt{ Best Practices}}).

We recommend the use of these options over passing options to Autoprefixer so that the config can be shared with other tools such as \href{https://github.com/babel/babel/tree/master/packages/babel-preset-env}{\texttt{ babel-\/preset-\/env}} and \href{https://stylelint.io/}{\texttt{ Stylelint}}.

Autoprefixer can be used to translate modern CSS Grid syntax into IE 10 and IE 11 syntax, but this polyfill will not work in 100\% of cases. This is why it is disabled by default.

First, you need to enable Grid prefixes by using either the {\ttfamily grid\+: \char`\"{}autoplace\char`\"{}} option or the {\ttfamily /$\ast$ autoprefixer grid\+: autoplace $\ast$/} control comment.

Second, you need to test every fix with Grid in IE. It is not an enable and forget feature, but it is still very useful. Financial \+Times and \+Yandex use it in production.

Third, there is only very limited auto placement support. Read the \href{\#grid-autoplacement-support-in-ie}{\texttt{ Grid Autoplacement support in IE}} section for more details.

Fourth, if you are not using the autoplacement feature, the best way to use Autoprefixer is by using {\ttfamily grid-\/template} or {\ttfamily grid-\/template-\/areas}.

There are three types of control comments\+:


\begin{DoxyItemize}
\item {\ttfamily /$\ast$ autoprefixer\+: (on$\vert$off) $\ast$/}\+: enable/disable all Autoprefixer translations for the whole block both {\itshape before} and {\itshape after} the comment.
\item {\ttfamily /$\ast$ autoprefixer\+: ignore next $\ast$/}\+: disable Autoprefixer only for the next property or next rule selector or at-\/rule parameters (but not rule/at‑rule body).
\item {\ttfamily /$\ast$ autoprefixer grid\+: (autoplace$\vert$no-\/autoplace$\vert$off) $\ast$/}\+: control how Autoprefixer handles grid translations for the whole block\+:
\begin{DoxyItemize}
\item {\ttfamily autoplace}\+: enable grid translations with autoplacement support.
\item {\ttfamily no-\/autoplace}\+: enable grid translations with autoplacement support {\itshape disabled} (alias for deprecated value {\ttfamily on}).
\item {\ttfamily off}\+: disable all grid translations.
\end{DoxyItemize}
\end{DoxyItemize}
