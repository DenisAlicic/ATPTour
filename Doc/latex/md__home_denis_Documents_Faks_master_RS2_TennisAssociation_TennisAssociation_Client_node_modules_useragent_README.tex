Useragent originated as port of \href{http://www.browserscope.org/}{\texttt{ browserscope.\+org}}\textquotesingle{}s user agent parser project also known as ua-\/parser. Useragent allows you to parse user agent string with high accuracy by using hand tuned dedicated regular expressions for browser matching. This database is needed to ensure that every browser is correctly parsed as every browser vendor implements it\textquotesingle{}s own user agent schema. This is why regular user agent parsers have major issues because they will most likely parse out the wrong browser name or confuse the render engine version with the actual version of the browser.

\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_useragent_README_autotoc_md3628}{}\doxysubsection{Agents, Operating\+System and Device instances}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_useragent_README_autotoc_md3628}
Most of the methods mentioned above return a Agent instance. The Agent exposes the parsed out information from the user agent strings. This allows us to extend the agent with more methods that do not necessarily need to be in the core agent instance, allowing us to expose a plugin interface for third party developers and at the same time create a uniform interface for all versioning.

The Agent has the following property


\begin{DoxyItemize}
\item {\ttfamily family} The browser family, or browser name, it defaults to Other.
\item {\ttfamily major} The major version number of the family, it defaults to 0.
\item {\ttfamily minor} The minor version number of the family, it defaults to 0.
\item {\ttfamily patch} The patch version number of the family, it defaults to 0.
\end{DoxyItemize}

In addition to the properties mentioned above, it also has 2 special properties, which are\+:


\begin{DoxyItemize}
\item {\ttfamily os} Operating\+System instance
\item {\ttfamily device} Device instance
\end{DoxyItemize}

When you access those 2 properties the agent will do on demand parsing of the Operating System or/and Device information.

The Operating\+System has the same properties as the Agent, for the Device we don\textquotesingle{}t have any versioning information available, so only the {\ttfamily family} property is set there. If we cannot find the family, they will default to {\ttfamily Other}.
