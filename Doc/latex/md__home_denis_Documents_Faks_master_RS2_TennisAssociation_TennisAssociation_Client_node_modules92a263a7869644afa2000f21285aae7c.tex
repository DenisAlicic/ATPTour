There are several prefixes to instructions that affect the way the work. Here is the list of them in the order of appearance in the function name\+:


\begin{DoxyItemize}
\item {\ttfamily i} -\/ perform operation in-\/place, storing the result in the host object (on which the method was invoked). Might be used to avoid number allocation costs
\item {\ttfamily u} -\/ unsigned, ignore the sign of operands when performing operation, or always return positive value. Second case applies to reduction operations like {\ttfamily mod()}. In such cases if the result will be negative -\/ modulo will be added to the result to make it positive
\end{DoxyItemize}\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules92a263a7869644afa2000f21285aae7c_autotoc_md31648}{}\doxysubsection{Postfixes}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules92a263a7869644afa2000f21285aae7c_autotoc_md31648}
The only available postfix at the moment is\+:


\begin{DoxyItemize}
\item {\ttfamily n} -\/ which means that the argument of the function must be a plain Java\+Script Number. Decimals are not supported.

When doing lots of reductions using the same modulo, it might be beneficial to use some tricks\+: like \href{https://en.wikipedia.org/wiki/Montgomery_modular_multiplication}{\texttt{ Montgomery multiplication}}, or using special algorithm for \href{https://en.wikipedia.org/wiki/Mersenne_prime}{\texttt{ Mersenne Prime}}.\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules92a263a7869644afa2000f21285aae7c_autotoc_md31656}{}\doxysubsection{Reduction context}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules92a263a7869644afa2000f21285aae7c_autotoc_md31656}
To enable this tricks one should create a reduction context\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var red = BN.red(num);}

\end{DoxyCode}
 where {\ttfamily num} is just a BN instance.

Or\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var red = BN.red(primeName);}

\end{DoxyCode}


Where {\ttfamily prime\+Name} is either of these \href{https://en.wikipedia.org/wiki/Mersenne_prime}{\texttt{ Mersenne Primes}}\+:


\begin{DoxyItemize}
\item `\textquotesingle{}k256'{\ttfamily  $\ast$}\textquotesingle{}p224\textquotesingle{}{\ttfamily  $\ast$}\textquotesingle{}p192\textquotesingle{}{\ttfamily  $\ast$}\textquotesingle{}p25519\textquotesingle{}\`{}
\end{DoxyItemize}

Or\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var red = BN.mont(num);}

\end{DoxyCode}


To reduce numbers with \href{https://en.wikipedia.org/wiki/Montgomery_modular_multiplication}{\texttt{ Montgomery trick}}. {\ttfamily .mont()} is generally faster than {\ttfamily .red(num)}, but slower than {\ttfamily BN.\+red(prime\+Name)}.\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules92a263a7869644afa2000f21285aae7c_autotoc_md31657}{}\doxysubsection{Converting numbers}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules92a263a7869644afa2000f21285aae7c_autotoc_md31657}
Before performing anything in reduction context -\/ numbers should be converted to it. Usually, this means that one should\+:


\begin{DoxyItemize}
\item Convert inputs to reducted ones
\item Operate on them in reduction context
\item Convert outputs back from the reduction context
\end{DoxyItemize}

Here is how one may convert numbers to {\ttfamily red}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var redA = a.toRed(red);}

\end{DoxyCode}
 Where {\ttfamily red} is a reduction context created using instructions above

Here is how to convert them back\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var a = redA.fromRed();}

\end{DoxyCode}
