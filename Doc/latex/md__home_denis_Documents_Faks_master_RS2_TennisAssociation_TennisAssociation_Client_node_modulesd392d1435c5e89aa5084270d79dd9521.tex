\begin{DoxyItemize}
\item If you want to change the default set of browsers, we recommend combining {\ttfamily last 2 versions}, {\ttfamily not dead} with a usage number like {\ttfamily $>$ 0.\+2\%}. This is because {\ttfamily last n versions} on its own does not add popular old versions, while only using a percentage above {\ttfamily 0.\+2\%} will in the long run make popular browsers even more popular. We might run into a monopoly and stagnation situation, as we had with Internet Explorer 6. Please use this setting with caution.
\item Select browsers directly ({\ttfamily last 2 Chrome versions}) only if you are making a web app for a kiosk with one browser. There are a lot of browsers on the market. If you are making general web app you should respect browsers diversity.
\item Don’t remove browsers just because you don’t know them. Opera Mini has 100 million users in Africa and it is more popular in the global market than Microsoft Edge. Chinese QQ Browsers has more market share than Firefox and desktop Safari combined.

{\ttfamily npx browserslist@latest -\/-\/update-\/db} updates {\ttfamily caniuse-\/lite} version in your npm, yarn or pnpm lock file.

You need to do it regularly for two reasons\+:


\begin{DoxyEnumerate}
\item To use the latest browser’s versions and statistics in queries like {\ttfamily last 2 versions} or {\ttfamily $>$1\%}. For example, if you created your project 2 years ago and did not update your dependencies, {\ttfamily last 1 version} will return 2 year old browsers.
\item {\ttfamily caiuse-\/lite} deduplication\+: to synchronize version in different tools.
\end{DoxyEnumerate}

\begin{quote}
What is deduplication? \end{quote}
Due to how npm architecture is setup, you may have a situation where you have multiple versions of a single dependency required.

Imagine you begin a project, and you add {\ttfamily autoprefixer} as a dependency. npm looks for the latest {\ttfamily caniuse-\/lite} version (1.\+0.\+30000700) and adds it to {\ttfamily package-\/lock.\+json} under {\ttfamily autoprefixer} dependencies.

A year later, you decide to add Babel. At this moment, we have a new version of {\ttfamily canuse-\/lite} (1.\+0.\+30000900). npm took the latest version and added it to your lock file under {\ttfamily @babel/preset-\/env} dependencies.

Now your lock file looks like this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{autoprefixer 7.1.4}
\DoxyCodeLine{  browserslist 3.1.1}
\DoxyCodeLine{    caniuse-\/lite 1.0.30000700}
\DoxyCodeLine{@babel/preset-\/env 7.10.0}
\DoxyCodeLine{  browserslist 4.13.0}
\DoxyCodeLine{    caniuse-\/lite 1.0.30000900}

\end{DoxyCode}


As you can see, we now have two versions of {\ttfamily caniuse-\/lite} installed.\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modulesd392d1435c5e89aa5084270d79dd9521_autotoc_md1883}{}\doxysection{Queries}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modulesd392d1435c5e89aa5084270d79dd9521_autotoc_md1883}
Browserslist will use browsers and Node.\+js versions query from one of these sources\+:


\begin{DoxyEnumerate}
\item {\ttfamily browserslist} key in {\ttfamily package.\+json} file in current or parent directories. {\bfseries{We recommend this way.}}
\item {\ttfamily .browserslistrc} config file in current or parent directories.
\item {\ttfamily browserslist} config file in current or parent directories.
\item {\ttfamily BROWSERSLIST} environment variable.
\item If the above methods did not produce a valid result Browserslist will use defaults\+: {\ttfamily $>$ 0.\+5\%, last 2 versions, Firefox ESR, not dead}.
\end{DoxyEnumerate}\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modulesd392d1435c5e89aa5084270d79dd9521_autotoc_md1884}{}\doxysubsection{Query Composition}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modulesd392d1435c5e89aa5084270d79dd9521_autotoc_md1884}
An {\ttfamily or} combiner can use the keyword {\ttfamily or} as well as {\ttfamily ,}. {\ttfamily last 1 version or $>$ 1\%} is equal to {\ttfamily last 1 version, $>$ 1\%}.

{\ttfamily and} query combinations are also supported to perform an intersection of all the previous queries\+: {\ttfamily last 1 version or chrome $>$ 75 and $>$ 1\%} will select ({\ttfamily browser last version} or {\ttfamily Chrome since 76}) and {\ttfamily more than 1\% marketshare}.

There is 3 different ways to combine queries as depicted below. First you start with a single query and then we combine the queries to get our final list.

Obviously you can {\itshape not} start with a {\ttfamily not} combiner, since there is no left-\/hand side query to combine it with. The left-\/hand is always resolved as {\ttfamily and} combiner even if {\ttfamily or} is used (this is an API implementation specificity).

You can specify the browser and Node.\+js versions by queries (case insensitive)\+:


\begin{DoxyItemize}
\item {\ttfamily defaults}\+: Browserslist’s default browsers ({\ttfamily $>$ 0.\+5\%, last 2 versions, Firefox ESR, not dead}).
\item {\ttfamily $>$ 5\%}\+: browsers versions selected by global usage statistics. {\ttfamily $>$=}, {\ttfamily $<$} and {\ttfamily $<$=} work too.
\begin{DoxyItemize}
\item {\ttfamily $>$ 5\% in US}\+: uses USA usage statistics. It accepts \href{https://en.wikipedia.org/wiki/ISO_3166-1_alpha-2\#Officially_assigned_code_elements}{\texttt{ two-\/letter country code}}.
\item {\ttfamily $>$ 5\% in alt-\/\+AS}\+: uses Asia region usage statistics. List of all region codes can be found at \href{https://github.com/ben-eb/caniuse-lite/tree/master/data/regions}{\texttt{ {\ttfamily caniuse-\/lite/data/regions}}}.
\item {\ttfamily $>$ 5\% in my stats}\+: uses \href{\#custom-usage-data}{\texttt{ custom usage data}}.
\item {\ttfamily $>$ 5\% in browserslist-\/config-\/mycompany stats}\+: uses \href{\#custom-usage-data}{\texttt{ custom usage data}} from {\ttfamily browserslist-\/config-\/mycompany/browserslist-\/stats.\+json}.
\item {\ttfamily cover 99.\+5\%}\+: most popular browsers that provide coverage.
\item {\ttfamily cover 99.\+5\% in US}\+: same as above, with \href{https://en.wikipedia.org/wiki/ISO_3166-1_alpha-2\#Officially_assigned_code_elements}{\texttt{ two-\/letter country code}}.
\item {\ttfamily cover 99.\+5\% in my stats}\+: uses \href{\#custom-usage-data}{\texttt{ custom usage data}}.
\end{DoxyItemize}
\item {\ttfamily dead}\+: browsers without official support or updates for 24 months. Right now it is {\ttfamily IE 10}, {\ttfamily IE\+\_\+\+Mob 11}, {\ttfamily Black\+Berry 10}, {\ttfamily Black\+Berry 7}, {\ttfamily Samsung 4} and {\ttfamily Opera\+Mobile 12.\+1}.
\item {\ttfamily last 2 versions}\+: the last 2 versions for {\itshape each} browser.
\begin{DoxyItemize}
\item {\ttfamily last 2 Chrome versions}\+: the last 2 versions of Chrome browser.
\item {\ttfamily last 2 major versions} or {\ttfamily last 2 i\+OS major versions}\+: all minor/patch releases of last 2 major versions.
\end{DoxyItemize}
\item {\ttfamily node 10} and {\ttfamily node 10.\+4}\+: selects latest Node.\+js {\ttfamily 10.\+x.\+x} or {\ttfamily 10.\+4.\+x} release.
\begin{DoxyItemize}
\item {\ttfamily current node}\+: Node.\+js version used by Browserslist right now.
\item {\ttfamily maintained node versions}\+: all Node.\+js versions, which are \href{https://github.com/nodejs/Release}{\texttt{ still maintained}} by Node.\+js Foundation.
\end{DoxyItemize}
\item {\ttfamily i\+OS 7}\+: the i\+OS browser version 7 directly.
\begin{DoxyItemize}
\item {\ttfamily Firefox $>$ 20}\+: versions of Firefox newer than 20. {\ttfamily $>$=}, {\ttfamily $<$} and {\ttfamily $<$=} work too. It also works with Node.\+js.
\item {\ttfamily ie 6-\/8}\+: selects an inclusive range of versions.
\item {\ttfamily Firefox ESR}\+: the latest \mbox{[}Firefox ESR\mbox{]} version.
\item {\ttfamily Phantom\+JS 2.\+1} and {\ttfamily Phantom\+JS 1.\+9}\+: selects Safari versions similar to Phantom\+JS runtime.
\end{DoxyItemize}
\item {\ttfamily extends browserslist-\/config-\/mycompany}\+: take queries from {\ttfamily browserslist-\/config-\/mycompany} npm package.
\item {\ttfamily supports es6-\/module}\+: browsers with support for specific features. {\ttfamily es6-\/module} here is the {\ttfamily feat} parameter at the URL of the \href{https://caniuse.com/}{\texttt{ Can I Use}} page. \item {\ttfamily since 2015} or {\ttfamily last 2 years}\+: all versions released since year 2015 (also {\ttfamily since 2015-\/03} and {\ttfamily since 2015-\/03-\/10}).
\item {\ttfamily unreleased versions} or {\ttfamily unreleased Chrome versions}\+: alpha and beta versions.
\item {\ttfamily not ie $<$= 8}\+: exclude browsers selected by previous queries.
\end{DoxyItemize}
