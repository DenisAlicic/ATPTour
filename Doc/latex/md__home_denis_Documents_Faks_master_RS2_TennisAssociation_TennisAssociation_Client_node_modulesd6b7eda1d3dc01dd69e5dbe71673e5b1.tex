{\ttfamily libnpmpublish} uses \href{https://npm.im/npm-registry-fetch}{\texttt{ {\ttfamily npm-\/registry-\/fetch}}}. Most options are passed through directly to that library, so please refer to \href{http://npm.im/npm-registry-fetch\#fetch-options}{\texttt{ its own {\ttfamily opts} documentation}} for options that can be passed in.

A couple of options of note\+:


\begin{DoxyItemize}
\item {\ttfamily opts.\+default\+Tag} -\/ registers the published package with the given tag, defaults to {\ttfamily latest}.
\item {\ttfamily opts.\+access} -\/ tells the registry whether this package should be published as public or restricted. Only applies to scoped packages, which default to restricted.
\item {\ttfamily opts.\+token} -\/ can be passed in and will be used as the authentication token for the registry. For other ways to pass in auth details, see the n-\/r-\/f docs.
\end{DoxyItemize}\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modulesd6b7eda1d3dc01dd69e5dbe71673e5b1_autotoc_md15020}{}\doxysubsubsection{$<$a name=\char`\"{}publish\char`\"{}$>$$<$/a$>$ $<$tt$>$$>$ libpub.\+publish(manifest, tar\+Data, \mbox{[}opts\mbox{]}) -\/$>$ Promise$<$/tt$>$}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modulesd6b7eda1d3dc01dd69e5dbe71673e5b1_autotoc_md15020}
Sends the package represented by the {\ttfamily manifest} and {\ttfamily tar\+Data} to the configured registry.

{\ttfamily manifest} should be the parsed {\ttfamily package.\+json} for the package being published (which can also be the manifest pulled from a packument, a git repo, tarball, etc.)

{\ttfamily tar\+Data} is a {\ttfamily Buffer} of the tarball being published.

If {\ttfamily opts.\+npm\+Version} is passed in, it will be used as the {\ttfamily \+\_\+npm\+Version} field in the outgoing packument. You may put your own user-\/agent string in there to identify your publishes.

If {\ttfamily opts.\+algorithms} is passed in, it should be an array of hashing algorithms to generate {\ttfamily integrity} hashes for. The default is `\mbox{[}\textquotesingle{}sha512'\mbox{]}{\ttfamily , which means you end up with}dist.\+integrity = \textquotesingle{}sha512-\/deadbeefbadc0ffee\textquotesingle{}\`{}. Any algorithm supported by your current node version is allowed -- npm clients that do not support those algorithms will simply ignore the unsupported hashes.\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modulesd6b7eda1d3dc01dd69e5dbe71673e5b1_autotoc_md15021}{}\doxyparagraph{Example}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modulesd6b7eda1d3dc01dd69e5dbe71673e5b1_autotoc_md15021}
