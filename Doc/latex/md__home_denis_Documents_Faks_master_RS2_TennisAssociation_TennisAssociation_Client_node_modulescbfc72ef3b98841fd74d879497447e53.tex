The following opts are used directly by {\ttfamily libnpmsearch} itself\+:


\begin{DoxyItemize}
\item {\ttfamily opts.\+limit} -\/ Number of results to limit the query to. Default\+: 20
\item {\ttfamily opts.\+from} -\/ Offset number for results. Used with {\ttfamily opts.\+limit} for pagination. Default\+: 0
\item {\ttfamily opts.\+detailed} -\/ If true, returns an object with {\ttfamily package}, {\ttfamily score}, and {\ttfamily search\+Score} fields, with {\ttfamily package} being what would usually be returned, and the other two containing details about how that package scored. Useful for UIs. Default\+: false
\item {\ttfamily opts.\+sort\+By} -\/ Used as a shorthand to set {\ttfamily opts.\+quality}, {\ttfamily opts.\+maintenance}, and {\ttfamily opts.\+popularity} with values that prioritize each one. Should be one of `\textquotesingle{}optimal'{\ttfamily ,}\textquotesingle{}quality\textquotesingle{}{\ttfamily ,}\textquotesingle{}maintenance\textquotesingle{}{\ttfamily , or}\textquotesingle{}popularity\textquotesingle{}{\ttfamily . Default\+:}\textquotesingle{}optimal\textquotesingle{}{\ttfamily  $\ast$}opts.\+maintenance$<$tt$>$-\/ Decimal number between0{\ttfamily and}1{\ttfamily that defines the weight of}maintenance{\ttfamily metrics when scoring and sorting packages. Default\+:}0.\+65{\ttfamily (same as}opts.\+sort\+By\+: \textquotesingle{}optimal\textquotesingle{}{\ttfamily ) $\ast$}opts.\+popularity$<$tt$>$-\/ Decimal number between0{\ttfamily and}1{\ttfamily that defines the weight of}popularity{\ttfamily metrics when scoring and sorting packages. Default\+:}0.\+98{\ttfamily (same as}opts.\+sort\+By\+: \textquotesingle{}optimal\textquotesingle{}{\ttfamily ) $\ast$}opts.\+quality$<$tt$>$-\/ Decimal number between0{\ttfamily and}1{\ttfamily that defines the weight of}quality{\ttfamily metrics when scoring and sorting packages. Default\+:}0.\+5{\ttfamily (same as}opts.\+sort\+By\+: \textquotesingle{}optimal\textquotesingle{}\`{})
\end{DoxyItemize}

{\ttfamily libnpmsearch} uses \href{https://npm.im/npm-registry-fetch}{\texttt{ {\ttfamily npm-\/registry-\/fetch}}}. Most options are passed through directly to that library, so please refer to \href{https://www.npmjs.com/package/npm-registry-fetch\#fetch-options}{\texttt{ its own {\ttfamily opts} documentation}} for options that can be passed in.

A couple of options of note for those in a hurry\+:


\begin{DoxyItemize}
\item {\ttfamily opts.\+token} -\/ can be passed in and will be used as the authentication token for the registry. For other ways to pass in auth details, see the n-\/r-\/f docs.
