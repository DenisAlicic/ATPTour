{\ttfamily npm link} is very similar to {\ttfamily npm install -\/g} except that instead of downloading the package from the repo, the just built {\ttfamily dist/@angular/cli/} folder becomes the global package. Additionally, this repository publishes several packages and we use special logic to load all of them on development setups.

Any changes to the files in the {\ttfamily angular-\/cli/} folder will immediately affect the global {\ttfamily @angular/cli} package, meaning that, in order to quickly test any changes you make to the cli project, you should simply just run {\ttfamily npm run build} again.

Now you can use {\ttfamily @angular/cli} via the command line\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{ng new foo}
\DoxyCodeLine{cd foo}
\DoxyCodeLine{npm link @angular/cli}
\DoxyCodeLine{ng serve}

\end{DoxyCode}


{\ttfamily npm link @angular/cli} is needed because by default the globally installed {\ttfamily @angular/cli} just loads the local {\ttfamily @angular/cli} from the project which was fetched remotely from npm. {\ttfamily npm link @angular/cli} symlinks the global {\ttfamily @angular/cli} package to the local {\ttfamily @angular/cli} package. Now the {\ttfamily angular-\/cli} you cloned before is in three places\+: The folder you cloned it into, npm\textquotesingle{}s folder where it stores global packages and the Angular CLI project you just created.

You can also use {\ttfamily ng new foo -\/-\/link-\/cli} to automatically link the {\ttfamily @angular/cli} package.

Please read the official \href{https://docs.npmjs.com/cli/link}{\texttt{ npm-\/link documentation}} and the \href{http://browsenpm.org/help\#linkinganynpmpackagelocally}{\texttt{ npm-\/link cheatsheet}} for more information.
