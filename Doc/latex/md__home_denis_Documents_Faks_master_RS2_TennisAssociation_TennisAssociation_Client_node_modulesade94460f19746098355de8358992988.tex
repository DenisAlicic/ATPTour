By default, {\ttfamily byte\+Size} converts the input number to a human readable string with metric units and a precision of 1.


\begin{DoxyCode}{0}
\DoxyCodeLine{> const byteSize = require('byte-\/size')}
\DoxyCodeLine{}
\DoxyCodeLine{> byteSize(1580)}
\DoxyCodeLine{\{ value: '1.6', unit: 'kB', long: 'kilobytes' \}}

\end{DoxyCode}


The object returned by {\ttfamily byte\+Size} defines a {\ttfamily to\+String} method therefore can be used directly in string context.


\begin{DoxyCode}{0}
\DoxyCodeLine{> `Filesize: \$\{byteSize(12400)\}`}
\DoxyCodeLine{'Filesize: 12.4 kB'}

\end{DoxyCode}


Override the default {\ttfamily to\+String} behaviour by setting \href{\#bytesizebytes-options--object-}{\texttt{ {\ttfamily options.\+to\+String\+Fn}}}.


\begin{DoxyCode}{0}
\DoxyCodeLine{> function toStringFn () \{}
\DoxyCodeLine{  return `**\$\{this.value\}\$\{this.unit\}**`}
\DoxyCodeLine{\}}
\DoxyCodeLine{}
\DoxyCodeLine{> `Filesize: \$\{byteSize(12400, \{ toStringFn \})\}`}
\DoxyCodeLine{'Filesize: **12.4kB**'}

\end{DoxyCode}


Beside the default of {\ttfamily metric}, there are three other built-\/in units available\+: {\ttfamily metric\+\_\+octet}, {\ttfamily iec} and {\ttfamily iec\+\_\+octet}.


\begin{DoxyCode}{0}
\DoxyCodeLine{> byteSize(1580, \{ units: 'iec' \})}
\DoxyCodeLine{\{ value: '1.5', unit: 'KiB', long: 'kibibytes' \}}
\DoxyCodeLine{}
\DoxyCodeLine{> byteSize(1580, \{ units: 'iec\_octet' \})}
\DoxyCodeLine{\{ value: '1.5', unit: 'Kio', long: 'kibioctets' \}}
\DoxyCodeLine{}
\DoxyCodeLine{> byteSize(1580, \{ units: 'metric\_octet' \})}
\DoxyCodeLine{\{ value: '1.6', unit: 'ko', long: 'kilooctets' \}}

\end{DoxyCode}


You can adjust the {\ttfamily precision}.


\begin{DoxyCode}{0}
\DoxyCodeLine{> byteSize(1580, \{ units: 'iec', precision: 3 \})}
\DoxyCodeLine{\{ value: '1.543', unit: 'KiB', long: 'kibibytes' \}}
\DoxyCodeLine{}
\DoxyCodeLine{> byteSize(1580, \{ units: 'iec', precision: 0 \})}
\DoxyCodeLine{\{ value: '2', unit: 'KiB', long: 'kibibytes' \}}

\end{DoxyCode}


Define custom units by passing an object containing one or more additional conversion tables to {\ttfamily options.\+custom\+Units}. In {\ttfamily options.\+units}, specify the name of a property from the {\ttfamily custom\+Units} object.


\begin{DoxyCode}{0}
\DoxyCodeLine{> const customUnits = \{}
\DoxyCodeLine{  simple: [}
\DoxyCodeLine{    \{ from: 0   , to: 1e3 , unit: ''  \},}
\DoxyCodeLine{    \{ from: 1e3 , to: 1e6 , unit: 'K', long: 'thousand' \},}
\DoxyCodeLine{    \{ from: 1e6 , to: 1e9 , unit: 'Mn', long: 'million' \},}
\DoxyCodeLine{    \{ from: 1e9 , to: 1e12, unit: 'Bn', long: 'billion' \}}
\DoxyCodeLine{  ]}
\DoxyCodeLine{\}}
\DoxyCodeLine{}
\DoxyCodeLine{> const \{ value, unit \} = byteSize(10000, \{ customUnits, units: 'simple' \})}
\DoxyCodeLine{}
\DoxyCodeLine{> `\$\{value\}\$\{unit\}`}
