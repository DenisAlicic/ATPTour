$<$/details$>$\hypertarget{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_nanomatch_README_autotoc_md10626}{}\doxysection{What is nanomatch?}\label{md__home_denis_Documents_Faks_master_RS2_TennisAssociation_TennisAssociation_Client_node_modules_nanomatch_README_autotoc_md10626}
Nanomatch is a fast and accurate glob matcher with full support for standard Bash glob features, including the following \char`\"{}metacharacters\char`\"{}\+: {\ttfamily $\ast$}, {\ttfamily $\ast$$\ast$}, {\ttfamily ?} and {\ttfamily \mbox{[}...\mbox{]}}.

\begin{DoxyItemize}

\item \href{\#features}{\texttt{ Features}}\+: jump to info about supported patterns, and a glob matching reference
\item \href{\#api}{\texttt{ API documentation}}\+: jump to available options and methods
\item \mbox{[}Unit tests\mbox{]}(test)\+: visit unit tests. there is no better way to learn a code library than spending time the unit tests. Nanomatch has 36,000 unit tests -\/ go become a glob matching ninja!
\end{DoxyItemize}

$<$details$>$

{\bfseries{How is this different?}}

{\bfseries{Speed and accuracy}}

Nanomatch uses \href{https://github.com/jonschlinkert/snapdragon}{\texttt{ snapdragon}} for parsing and compiling globs, which results in\+:


\begin{DoxyItemize}
\item Granular control over the entire conversion process in a way that is easy to understand, reason about, and customize.
\item Faster matching, from a combination of optimized glob patterns and (optional) caching.
\item Much greater accuracy than minimatch. In fact, nanomatch passes {\itshape all of the spec tests} from bash, including some that bash still fails. However, since there is no real specification for globs, if you encounter a pattern that yields unexpected match results \href{../../issues}{\texttt{ after researching previous issues}}, \href{../../issues/new}{\texttt{ please let us know}}.
\end{DoxyItemize}

{\bfseries{Basic globbing only}}

Nanomatch supports \href{\#features}{\texttt{ basic globbing only}}, which is limited to {\ttfamily $\ast$}, {\ttfamily $\ast$$\ast$}, {\ttfamily ?} and regex-\/like brackets.

If you need support for the other \href{\#bash-expansion-libs}{\texttt{ bash \char`\"{}expansion\char`\"{} types}} (in addition to the wildcard matching provided by nanomatch), consider using \href{https://github.com/micromatch/micromatch}{\texttt{ micromatch}} instead. \+\_\+(micromatch $>$=3.\+0.\+0 uses the nanomatch parser and compiler for basic glob matching)\+\_\+

{\itshape Backslashes and quotes} can be used to escape characters, forcing nanomatch to regard those characters as a literal characters.

{\bfseries{Backslashes}}

Use backslashes to escape single characters. For example, the following pattern would match {\ttfamily foo/$\ast$/bar} exactly\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{'foo/\(\backslash\)*/bar'}

\end{DoxyCode}


The following pattern would match {\ttfamily foo/} followed by a literal {\ttfamily $\ast$}, followed by zero or more of any characters besides {\ttfamily /}, followed by {\ttfamily /bar}.


\begin{DoxyCode}{0}
\DoxyCodeLine{'foo/\(\backslash\)**/bar'}

\end{DoxyCode}


{\bfseries{Quoted strings}}

Use single or double quotes to escape sequences of characters. For example, the following patterns would match {\ttfamily foo/$\ast$$\ast$/bar} exactly\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{'foo/"{}**"{}/bar'}
\DoxyCodeLine{'foo/\(\backslash\)'**\(\backslash\)'/bar'}
\DoxyCodeLine{"{}foo/'**'/bar"{}}

\end{DoxyCode}


{\bfseries{Matching literal quotes}}

If you need to match quotes literally, you can escape them as well. For example, the following will match {\ttfamily foo/\char`\"{}$\ast$\char`\"{}/bar}, {\ttfamily foo/\char`\"{}a\char`\"{}/bar}, {\ttfamily foo/\char`\"{}b\char`\"{}/bar}, or {\ttfamily foo/\char`\"{}c\char`\"{}/bar}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{'foo/\(\backslash\)\(\backslash\)"{}*\(\backslash\)\(\backslash\)"{}/bar'}

\end{DoxyCode}


And the following will match `foo/'$\ast$\textquotesingle{}/bar{\ttfamily ,}foo/\textquotesingle{}a\textquotesingle{}/bar{\ttfamily ,}foo/\textquotesingle{}b\textquotesingle{}/bar{\ttfamily , or}foo/\textquotesingle{}c\textquotesingle{}/bar\`{}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{'foo/\(\backslash\)\(\backslash\)\(\backslash\)'*\(\backslash\)\(\backslash\)\(\backslash\)'/bar'}

\end{DoxyCode}
