\item (\href{https://github.com/angular/protractor/commit/27b485019589cd662ee69e7920893ffa50774b97}{\texttt{ 27b4850}}) Support {\ttfamily SELENIUM\+\_\+\+PROMISE\+\_\+\+MANAGER=0} (\#72)

There are three major ways this was done in this change\+:
\begin{DoxyItemize}
\item In {\ttfamily call\+When\+Idle}, if {\ttfamily flow.\+is\+Idle} is not defined, we assume we are working with a {\ttfamily Simple\+Scheduler} instance, and so the flow is effectively idle.
\item In {\ttfamily init\+Jasmine\+Wd}, if {\ttfamily flow.\+reset} is not defined, we assume we are working with a {\ttfamily Simple\+Scheduler} instance, and so don\textquotesingle{}t bother resetting the flow.
\item In {\ttfamily wrap\+In\+Control\+Flow}, we use {\ttfamily flow.\+promise} to create a new promise if possible. Since {\ttfamily new webdriver.\+promise.\+Promise()} would have always made a {\ttfamily Managed\+Promise}, but {\ttfamily flow.\+promise} will do the right thing.
\item In {\ttfamily wrap\+Compare}, we avoid the webdriver library entirely, and never instance any extra promises. Using {\ttfamily webdriver.\+promise.\+when} and {\ttfamily webdriver.\+promise.\+all} could have been a problem if our instance of {\ttfamily webdriver} had the control flow turned on, but another instance somewhere did not (or even the same instance, but just at a different point in time). Instead we use the new {\ttfamily maybe\+Promise} tool, which is a mess but is also exactly what we want.
\item In {\ttfamily specs/$\ast$}, we replace {\ttfamily webdriver.\+promise.\+fulfilled} with {\ttfamily webdriver.\+promise.\+when}.
\item In {\ttfamily specs/$\ast$}, a new version of {\ttfamily adapter\+Spec.\+js} and {\ttfamily error\+Spec.\+js} are created\+: {\ttfamily async\+Await\+Adapter\+Spec.\+ts} and {\ttfamily async\+Await\+Error\+Spec.\+ts}.
\end{DoxyItemize}

I also also fixed a minor bug where we weren\textquotesingle{}t correctly checking for promises inside an array of expected results. Before we had\+:
\end{DoxyItemize}


\begin{DoxyCode}{0}
\DoxyCodeLine{expected = Array.prototype.slice.call(arguments, 0);}
\DoxyCodeLine{}
\DoxyCodeLine{...}
\DoxyCodeLine{}
\DoxyCodeLine{webdriver.promise.isPromise(expected);}

\end{DoxyCode}


I thought about it for a little while, and there\textquotesingle{}s no way that\textquotesingle{}s correct. {\ttfamily expected} is an {\ttfamily Array$<$any$>$}, there\textquotesingle{}s no way it has a {\ttfamily .then} function.
